Sub-Action Routine Architectures

How does the AI brain decide where an NPC goes within a location? Three approaches compared.

A) Role-Based Templates

Explicit scripts per (AddressType, Role) pair. The template IS the routine.

Architecture
AI Brain Processing
// Dave arrives at Office, 7:30 AM
template = Registry.Get("Office", "Employee")
schedule = template.Generate(address, rng)
// Template produces a fixed sequence
// PersonBehavior just follows it step by step

// Problem: New location type "CoworkingSpace"
// Need: CoworkingEmployeeTemplate
// Need: CoworkingVisitorTemplate
// Need: CoworkingCleanerTemplate
// Each one manually authored

// Problem: Office doesn't have a "Break Room"
// (small office, no break room generated)
// Template step 4 → crash or skip?
// Need fallback logic per template step

// Problem: Killer at SuburbanHome
// Need: SuburbanHomeKillerTemplate
// But killer behavior varies wildly...
// Searching for victim? Lying in wait?
// Template per killer strategy?
Templates needed for 6 types × ~4 roles:
~24 templates to author
Each one is a bespoke script. Grows linearly with types × roles.

B) Goal-Driven with Room Tags

Rooms have tags. NPCs have needs. The brain finds rooms that satisfy needs.

Architecture
AI Brain Processing
// Dave arrives at Office, 7:30 AM
needs = Action("Work").GetNeeds()
// → primary: work_area, periodic: [food, restroom]
room = address.FindByTag("entrance")
// → Lobby (always enter via entrance first)
moveTo(Lobby)
room = address.FindByTag("work_area")
// → Cubicle Area
pathfind(Lobby → Elevator → Cubicle Area)

// 9:45 AM — food need triggers
room = address.FindByTag("food")
// → Break Room
pathfind(Cubicle Area → Break Room)

// New location type "CoworkingSpace"?
// Just tag its rooms! No new templates.
// Hot Desk: [work_area]
// Kitchen: [food, social]
// Dave's brain works here automatically.

// Edge case: small office, no break room
// FindByTag("food") → null
// Need goes unsatisfied. NPC stays put.
// Realistic! Not every office has a break room.

// Edge case: Killer at SuburbanHome
// Action("KillPerson").GetNeeds()
// → primary: find_target (find room with victim)
// But HOW to enter? Front door? Window?
// Needs don't capture strategy...
// "find [entrance]" gives Front Door.
// A killer wouldn't use the front door.
// Need something more than tags here?

C) Hybrid: Tagged Rooms + Rhythm Templates

Templates define the pattern of behavior. Tags resolve which room fulfills each step.

Architecture
Key: Rhythms are reusable across location types
"WorkDay" rhythm works at Office, CoworkingSpace, Factory — anywhere with [work_area] + [entrance] tags.
"Patronize" rhythm works at Diner, DiveBar, Cafe — anywhere with [service_area] + [entrance].
"Inhabit" rhythm works at SuburbanHome, ApartmentBuilding — anywhere with [bedroom] + [kitchen] + [living].
AI Brain Processing
// Dave arrives at Office, 7:30 AM
rhythm = Rhythms.Get("WorkDay")
// Same rhythm for any workplace
step1 = rhythm.Next() → "enter via [entrance]"
room = address.FindByTag("entrance") → Lobby
path = pathfind(Road → Lobby)
moveTo(Lobby)

step2 = rhythm.Next() → "go to [work_area]"
room = address.FindByTag("work_area")
// → Cubicle Area
path = pathfind(Lobby → Elevator → Cubicle)
// Pathfinder discovers intermediate rooms!
moveTo(Elevator) → moveTo(Cubicle Area)

// 9:45 AM — rhythm fires periodic [food]
room = address.FindByTag("food") → Break Room
pathfind(Cubicle → Break Room) → direct

// New location "CoworkingSpace"?
// Tag rooms, reuse "WorkDay" rhythm. Done.

// Small office, no break room?
// FindByTag("food") → null
// Rhythm skips step. Realistic.

// KILLER at SuburbanHome:
// Uses rhythm "Intrude", not "WorkDay"
rhythm = Rhythms.Get("Intrude")
step1 → "enter via [covert_entry]"
room = address.FindByTag("covert_entry")
// → Back Window, Back Door, etc.
step2 → "find [occupied_by: target]"
// → Bedroom (victim is sleeping)
step3 → "execute action in current room"
// Traces: fingerprints on window, in hallway,
// on bedroom door — all from pathfinding

// Bartender at DiveBar?
// Uses rhythm "StaffShift"
step1 → "enter via [staff_entry]" → Back Door
step2 → "go to [work_area]" → Bar Area
// Different entry than patron, same tags
Rhythms needed for all behaviors:
~6-8 rhythms total
WorkDay, Patronize, Inhabit, StaffShift, Intrude, Socialize, Sleep, Visit
Each works across ALL location types via tags. Grows with behaviors, not locations.

Comparison Summary

A) Role Templates B) Tag-Driven C) Hybrid
Scales with new location types Poorly — new templates per type × role Well — just tag rooms Well — just tag rooms
Handles missing rooms Crash or manual fallback Graceful — need unsatisfied Graceful — step skipped
Behavioral variety High — fully scripted Medium — needs are generic High — rhythm adds personality
Killer/special behavior Bespoke template per strategy Hard — needs don't capture strategy "Intrude" rhythm + [covert_entry] tag
Intermediate room traces Only if scripted per step Via pathfinding Via pathfinding
Things to author ~24 templates (types × roles) Tag vocabulary + need definitions ~6-8 rhythms + tag vocabulary
Pathfinding built in No — rooms listed explicitly Yes — find room, pathfind to it Yes — find room, pathfind to it
