Outdoor Space Connection Graphs

Does the "zones connected by passages" model work for outdoor locations? Each box is a sublocation, each line is a connection with its access type.

City Park

Most connections are open — you walk between zones freely. Restrooms have a door. Wooded area provides concealment (line-of-sight blocked).

Parking Garage (4-story)

Vertical structure — stairwell is a single node per floor connected vertically by stairs. Ramp connects parking areas between floors. Entry booth has a gate.

Hiking Trail Area

Graph is a tree with a loop trail. Hidden cave only discoverable via investigation — not visible to NPCs on normal routes. Connections have travel time weight (a trail segment takes minutes, not seconds).

Campground

Campsite clusters as single zones vs. individual sites is a depth question. Gate at entrance matters for access control (after-hours). Bathhouse has a door (indoor sublocation within an outdoor location).

Observations

  • The tree-from-Road model works for all of these — even loops (hiking trail) are just extra edges on the tree
  • Connection types vary: open passage, path/trail, door, gate, ramp, stairs, hidden path, elevator
  • Some connections are one-directional or restricted: hidden paths only discoverable via investigation, gates that lock at night
  • Depth varies by location type: a park is 2-3 levels deep, a hiking area can be 4+
  • Indoor pockets within outdoor spaces: restrooms, camp office, hidden cave — same model, just a "door" connection
  • Granularity is a design choice: "Sites A1-A6" as one zone vs. six separate sublocations
