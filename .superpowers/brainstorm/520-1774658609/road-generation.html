Road Network Generation

How should roads be laid out to form city blocks?

A: Full-Span Grid

Every road runs the entire width or height of the map. Perfectly regular grid creates uniform rectangular blocks. Simple to generate — just pick spacing for each road.

Pros

  • Simplest algorithm
  • All roads guaranteed connected
  • Very predictable block shapes

Cons

  • Looks like graph paper
  • No dead ends, no character
  • Every block perfectly rectangular

B: Hierarchical Grid

Major arterial roads span the full map creating large super-blocks. Secondary roads subdivide those blocks — more subdivisions in urban center, fewer at edges. Suburban blocks stay large; urban blocks get chopped small.

Pros

  • Natural urban/suburban density difference
  • Major roads give navigational landmarks
  • Block size varies organically
  • Still guaranteed connected (arterials span full map)

Cons

  • Slightly more complex generation
  • Need to decide subdivision rules

C: Organic / Partial Roads

Roads don't all run the full width/height. Some terminate, some offset, creating irregular block shapes and T-intersections. More realistic but harder to guarantee connectivity.

Pros

  • Most realistic-looking
  • Interesting irregular blocks
  • T-intersections add character

Cons

  • Hardest to generate correctly
  • Connectivity not guaranteed — needs validation
  • Addressing gets complex
  • May create unusable block shapes
